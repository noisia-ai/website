# 36 — Decision register

## Decisiones definitivas

| ID | Decisión | Motivo |
|---|---|---|
| D01 | Signal Pulse es táctico de Marketing | La puerta comercial actual son Marketing, agencias y data intelligence |
| D02 | No adaptar T&B | Sería mucha vuelta y generaría output equivocado |
| D03 | Pipeline propio | El reporte necesita contrato, charts y prompts propios |
| D04 | Corpus vivo | Evitar snapshot obsoleto y permitir live/published/diff |
| D05 | Periodización mensual | Detectar nacimiento, deltas, longevidad, persistencia y decay |
| D06 | Navegación de 11 pantallas | Separar pantallas reales de componentes |
| D07 | Overview chart-first | Marketers necesitan visuales para justificar decisiones |
| D08 | Charts social listening reinventados | Familiaridad + diferenciación Noisia |
| D09 | Source Wizard | Integrar nuevas fuentes sin complejidad ad hoc |
| D10 | Tipos de fuente | No todo es mention; hay performance/entity/knowledge |
| D11 | Composer central | IA detecta, Noisia cura, publish guarda corte editorial |
| D12 | Pausar nuevas metodologías | Enfocar rama y evitar dispersión |

| O01 | ¿Semantic Galaxy con D3, SVG o Canvas? | Prototipo rápido con SVG si N bajo; Canvas si escala |

## Anti-decisiones

No hacer:

- dashboard genérico de social listening;
- mega lista de tabs;
- output en forma de deck largo;
- cards largas;
- copiar wordcloud literal;
- usar sentimiento de pastel;
- esconder fuentes y limitaciones;
- mezclar data live con published sin aviso.
