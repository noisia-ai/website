# 32 — Product analytics and feedback loop

## Propósito

Medir si Signal Pulse realmente ayuda a Marketing a decidir y justificar acciones.

## Métricas de uso

| Métrica | Qué indica |
|---|---|
| report_views | consumo general |
| screen_views | qué pantallas importan |
| chart_interactions | valor de visualizaciones |
| evidence_drawer_opens | necesidad de prueba |
| marketing_move_clicks | accionabilidad |
| filters_used | exploración real |
| exports/shares | uso en reuniones |
| composer_edits | carga editorial |
| published_cuts | cadencia operativa |

## Métricas de calidad

| Métrica | Target conceptual |
|---|---|
| % señales con evidencia suficiente | alto |
| % moves con owner sugerido | alto |
| % charts con data_ref válido | 100% |
| % published cuts sin blockers | 100% |
| warnings por reporte | monitorear |
| source freshness | alto |
| period comparability | alto |
| rework de copy | bajar con prompts mejores |

## Métricas de valor para Marketing

Recolectar feedback cualitativo:

- ¿Qué señal usarías en campaña?
- ¿Qué chart usarías para justificar presupuesto?
- ¿Qué movimiento sí ejecutarías?
- ¿Qué recomendación fue demasiado obvia?
- ¿Qué parte no entendiste?
- ¿Qué faltó para llevarlo a una reunión?

## Composer analytics

Medir:

- señales generadas vs promovidas;
- señales ocultas por ruido;
- señales fusionadas;
- edits por señal;
- moves eliminados;
- copy rewrite rate;
- gates que más fallan;
- tiempo de publish.

## Signal lifecycle feedback

Aprender:

- señales que se promovieron y después crecieron;
- señales que parecían importantes y murieron;
- movimientos que se ejecutaron;
- charts que sí llevaron a acción;
- fuentes más útiles por industria.

## Regla final

Signal Pulse debe mejorar con uso. Las decisiones editoriales y feedback de Marketing alimentan el sistema, no solo los prompts.
